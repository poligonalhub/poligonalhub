Tipos de Variáveis

Boolean Vermelho Escuro. Armazena um valor de Verdadeiro ou Falso (True/False). Muito usada para condições em Branch. Verificar se o jogador possui a chave (HasKey?) antes de abrir a porta.
Integer Verde Água (Cyan). Representa números inteiros, ou seja, números sem partes decimais (ex: 1, 5, -10). Útil para contadores, pontuações, quantidades de itens. Contar a quantidade de vidas restantes do jogador (Lives = 3).
Float Verde Limão (Light Green). Representa números com partes decimais, ou frações (ex: 3.14, 0.5, -2.7). Comumente usado para velocidade, saúde, distância. Barra de saúde do jogador (Health = 95.5).
String Magenta (Rosa Escuro). Usado para armazenar textos (ex: "Olá", "Menu Principal"). Diferente do Text, não é localizado para diferentes idiomas. Armazenar o nome de usuário digitado no login (PlayerName = "HeroiJP").
Transform Laranja. É uma combinação de Vector (Localização), Rotator (Rotação) e Vector (Escala). Define a posição, rotação e tamanho de um ator no mundo. Definir o local exato, para onde o personagem olha e o tamanho de uma magia sendo conjurada.
Vector Amarelo/Ouro. Composto por três Floats (X, Y, Z). Geralmente usado para representar pontos no espaço 3D, direções, e forças físicas. Aplicar um impulso para empurrar um objeto na direção X (X=500, Y=0, Z=0).
Rotator Roxo/Lilás. Composto por três valores de ângulo (Pitch, Yaw, Roll). Representa a rotação de um ator em relação aos três eixos do mundo. Fazer a cabeça do inimigo girar para seguir o jogador (Yaw = 90).

Eventos do Actor

Event BeginPlay Ocorre quando o jogo começa para este Actor. É o ponto de partida ideal para inicializar variáveis e realizar ações de setup iniciais. Tocar uma animação de spawn e encher a barra de stamina do monstro assim que ele surge.
Event Tick É chamado a cada frame gerado na tela. A quantidade de vezes por segundo depende do FPS do jogo. Use com cuidado, pois lógicas pesadas no Tick reduzem a performance do jogo. Rotacionar constantemente as pás de um ventilador ou moinho de vento.
Event ActorBeginOverlap Disparado quando a colisão (ou trigger) deste Actor começa a sobrepor a colisão de outro Actor especificado. Ex: Entrar numa área para abrir a porta. O jogador encosta em uma poção caída no chão para recupera a vida.
Event ActorEndOverlap Disparado no exato momento em que dois Actors deixam de se sobrepor/tocar. Ex: Sair da área de abrir a porta. O jogador sai da área segura, acendendo o aviso de perigo na tela.
Event AnyDamage Gatilho geral para dano. Chamado sempre que o Actor recebe dano, não importando a origem. Passa informações como quantidade de dano e quem o causou. Subtrair vida, independentemente se o jogador pisou na lava ou tomou um tiro.
Event PointDamage Representa dano direcional causado por projéteis, armas hit-scan ou ataques corpo-a-corpo (melee). Se o hit-scan atingir o osso da cabeça, aplicar multiplicador de Headshot (x2).
Event RadialDamage Chamado quando o Actor recebe dano em área. Ex: Dano de explosão indireta. Granadas que machucam todos os inimigos no raio de 500 unidades.
Event EndPlay Notifica que este Actor está sendo deletado, destruído ou o nível está sendo encerrado. Salvar a posição do jogador no Checkpoint antes dele sumir ao sair da fase.
Event Destroyed Chamado quando o Actor é destruído. (Nota: Obsoleto, prefira usar o Event EndPlay). Espalhar partículas de madeira voando quando um barril for destruído.
Event ActorBeginCursorOver Disparado quando a interface de mouse é usada e o cursor entra nos limites físicos do Actor. Aplicar um outline brilhante no item do cenário quando passa o mouse por cima (RPG isométrico).

Atalhos Úteis (Blueprints)

Atalhos Rápidos de Nós Segure a tecla correspondente e faça um Left Click na área livre do gráfico para spawnar o nó instantaneamente:

B : Branch
S : Sequence
D : Delay
O : Do Once
F : ForEachLoop
M : MultiGate Em vez de buscar por "Branch" na barra de busca, apenas segure B + clique para economizar tempo.
Gerenciar Conexões Alt + Left Click: Quebra (desconecta) os fios de um pino específico.
Ctrl + Drag: Move todos os fios de um pino para outro. Errou a conexão de uma variável? Segure Alt e clique nela para apagá-la sem deletar o nó inteiro.
Organização e Comentários Selecione vários nós e pressione C para envolvê-los em uma caixa de Comentário.
Selecione dois nós ou mais e pressione Q para alinhar seus nós em linha reta. Usar 'C' para escrever "Sistema de Pulo" envolvendo todos os blocos relacionados ao pulo.

Controle de Fluxo

Branch Avalia uma condição Booleana e redireciona o fluxo de execução para "True" ou "False". Equivale ao "if" na programação tradicional. O jogador tem dinheiro suficiente? Se True, desconta o dinheiro e entrega o item; Se False, exibe mensagem "Sem ouro".
Do N Permite que um trecho de lógica seja executado exatamente "N" vezes. Após atingir N, ele bloqueia até que seja resetado através do pino de entrada "Reset". Uma arma que pode atirar apenas 3 rajadas (N=3) antes de quebrar permanentemente ou precisar recarregar.
DoOnce Garante que a lógica conectada a ele seja executada apenas uma única vez, a menos que receba um sinal no pino de entrada "Reset". Disparar um áudio de tutorial apenas na primeira vez que o jogador tentar pular.
Flip Flop Alterna a execução entre duas saídas diferentes (A e B) cada vez que é chamado. Ótimo para interruptores do tipo Liga/Desliga. Apertar "F" liga a lanterna (A). Apertar "F" de novo desliga a lanterna (B).
For Loop Executa um bloco de ações repetidamente usando um índice que vai do número inicial até o número final definido. Ele finaliza na saída "Completed". Spawna 10 moedas em posições sequenciais no chão quando um baú for aberto.
For Loop with Break Idêntico ao For Loop, porém possui um pino de entrada "Break", que permite interromper a repetição prematuramente se uma condição for satisfeita. Procurar por uma Poção de Vida numa mochila de 20 slots. Interromper (Break) assim que achar, para poupar processamento.
Gate Funciona como um portão. O fluxo entra por "Enter", mas só passa para "Exit" se o portão estiver no estado aberto (Open). Pode ser aberto, fechado e alternado dinamicamente. Um lança-chamas que só atira (Enter) se o botão do mouse estiver pressionado abrindo a válvula (Open) e para se soltar (Close).
MultiGate Possui uma entrada e múltiplas saídas numeradas. A cada vez que é acionado, ativa uma saída diferente de forma sequencial ou aleatória, dependendo das configurações. Fazer o NPC soltar um resmungo aleatório ou diferente cada vez que o jogador interagir repetidamente.
Retriggerable Delay Atrasa a execução. Porém, se for chamado novamente antes do tempo acabar, o contador é reiniciado e ele começa a contar do zero. Manter o hud de vida visível na tela; se o jogador tomar dano de novo, zera a contagem e espera mais 5 segundos antes de escondê-lo.
Sequence Recebe um fluxo de execução e dispara os pinos de saída em ordem (0, depois 1, depois 2...). Útil para organizar o gráfico e evitar fios longos, pois executa os blocos na mesma frame. Ao morrer, primeiro (0) toca a animação de morte, depois (1) atualiza a contagem de vidas, depois (2) mostra a tela de Game Over.
While Loop Executa a saída de loop repetidamente enquanto a condição Booleana acoplada for verdadeira (True). Cuidado para não gerar loops infinitos que travam o motor. Fazer cálculos matemáticos complexos até alcançar o resultado desejado de uma distância de navegação da Inteligência Artificial.
